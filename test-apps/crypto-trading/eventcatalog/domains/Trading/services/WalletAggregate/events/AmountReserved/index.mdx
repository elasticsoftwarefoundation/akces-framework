## Overview

<Admonition type="info">
  This event is emitted when funds are temporarily reserved in a user's wallet, typically for a pending trade or transaction.
</Admonition>

The `AmountReservedEvent` indicates that a specific amount of cryptocurrency has been reserved in a user's wallet. Reserving funds ensures they cannot be used for other transactions until the pending operation completes or is cancelled.

## Event Structure

| Field | Type | Description |
|-------|------|-------------|
| `userId` | String | The user ID that owns the wallet (aggregate identifier) |
| `currency` | String | The type of (crypto)currency being reserved (e.g., "BTC", "ETH") |
| `amount` | BigDecimal | The amount of currency being reserved |
| `referenceId` | String | The unique identifier for the reservation, typically referencing an order or transaction ID |

## Producers

This event is produced by:
- **WalletAggregate**: When handling a reserve funds command

## Consumers

This event is consumed by:
- **OrderProcessManager**: To track fund reservations for pending orders
- **AccountingService**: For financial reporting and auditing purposes
